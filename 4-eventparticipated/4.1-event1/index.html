Event 1

 Event Name: Meet up 30/5

 Date & Time: 09:00, May 30, 2026

 Location: 26th Floor, Bitexco Tower, 02 Hai Trieu Street, Saigon Ward, Ho Chi Minh City

 Role: Attendee

Event Objectives

  • Share best practices in modern application design and cloud architecture optimization[cite: 1, 5].
  • Introduce Domain-Driven Design (DDD) methodologies, event-driven architectures, and compute placement blueprints[cite: 1].
  • Provide actionable roadmaps for secure AWS hands-on practice using open-source local emulators and GenAI tools[cite: 1, 3].
  • Equip participants with enterprise systemic mindsets (DevOps Foundations) and personal growth mechanics (overcoming procrastination, building technical presentation confidence)[cite: 2, 4, 5].

Speakers

  • Huynh An Khuong, Mai Quoc Anh, Nguyen Tran Minh Quan - FCAJ Challenger Team (Hackathon Operational Experience Sharing)[cite: 1].
  • Nguyen Thi Quynh Nhu - FCAJ Member (Confidence Architecture Presentation)[cite: 2].
  • Huynh Thai Linh - FCAJ Member (AWS Cloud Quest & Floci Optimization Topic)[cite: 3].
  • Khac Uy Pham - 3rd Year Student, VGU (Deconstructing the Procrastination Iceberg)[cite: 4].
  • Tran Minh Quân - Systems Engineer (The Hidden Iceberg of a Project: DevOps Foundations)[cite: 5].

Key Highlights

1. Tactical Insights from Hackathon Competitions

  • The Essence of Hackathons: High-intensity, fast-paced engineering events focused on turning conceptual theories into functional prototypes or MVPs within tight 24-to-48-hour sprints to tackle real-world pain points[cite: 1].
  • Case Studies from Active Projects:
    • SynthHunter: An AI-driven audio verification classification system utilizing a three-pillar detection engine (Speech Dynamics, Encoder Behavior, Temporal Rhythm) to mitigate deepfake fraud and compliance risks[cite: 1].
    • Vortex: An interconnected automated pipeline managing the full recruitment journey from initial CV parsing to interactive behavioral and technical interview simulations[cite: 1].
  • Key Takeaways: Always start by solving a real-world problem, experiment relentlessly, utilize modern tooling structures fully, and accept operational stresses (back pains, sleep deprivation, last-minute errors) as vital parts of engineering growth[cite: 1].

2. Soft Skills & Confidence Architecture for Engineers

  • Redefining Confidence: True confidence is not a loud ego or flawless perfectionism that claims to have every answer; it is the courage to initiate action and step forward even when experiencing nervousness and anxiety[cite: 2].
  • The Core Competency Bridge: While deep technical skills and coding serve as the critical foundation, professional communication and confidence act as the bridge that brings your innovations into production reality[cite: 2].
  • Confidence Hacking Frameworks: Utilizing the 5-Second Rule to execute ideas before cognitive self-doubt triggers, preparing thoroughly to establish operational Clarity, and building self-trust by celebrating Small Wins[cite: 2].

3. Optimized AWS Learning Roadmaps & Open-Source Floci Emulator

  • Beginner Cloud Friction Points: Entry-level cloud practitioners struggle with continuous AWS bill anxiety, forgetting to destroy active test resources, or exceeding Free Tier capacity limits[cite: 3].
  • Floci Architecture: Introducing an ultra-lightweight open-source AWS Local Emulator (24ms startup time, 13MiB idle memory footprints, yet supporting 50+ core services including EC2, Lambda, S3, DynamoDB, SQS, and API Gateway), outperforming the standard free LocalStack Community tier[cite: 3].
  • 3-Phase Practical Learning Journey:
    • Phase 1 (Mind & Architecture): Engaging in guided, gamified cloud quests via AWS Cloud Quest[cite: 3].
    • Phase 2 (Code & Fast Testing): Building and verifying microservices locally with zero cost using Floci[cite: 3].
    • Phase 3 (Real Deployment): Launching production-ready applications onto the official AWS Cloud fabric[cite: 3].

4. Deconstructing Procrastination via the Iceberg Model

  • Surface vs. Hidden Root Causes: Surface-level indicators like laziness and social media distractions account for only 20-30%. The true underlying cause (70-80%) stems from emotional regulation difficulties when facing stressful tasks, not poor time management[cite: 4].
  • The Three Invisible Forces of Fear:
    • Fear of Inadequacy: When encountering broad technical roadmaps (Kubernetes, Terraform), the brain closes the tab to avoid cognitive overwhelm[cite: 4].
    • Fear of Judgment: Reluctance to push working code to public GitHub repositories due to fear of external evaluation[cite: 4].
    • Fear of Failure: Utilizing procrastination as a defense mechanism (“I failed because I didn’t try hard, not because I lack skill”)[cite: 4].
  • Remediation Techniques: Move away from self-blame, practice explicit “Fear Naming”, and implement the 5-Minute Rule to override initial inertia and allow momentum to take control[cite: 4].

5. Corporate DevOps Foundations & Systems Thinking

  • The Project Management Iceberg: Visible symptoms like missed deadlines, production bugs, and team burnout originate from deep structural defects: ambiguous requirements, siloed communication blocks, manual processes, and low ownership[cite: 5].
  • The Essence of DevOps: DevOps extends far beyond basic tools (Docker, Kubernetes, CI/CD pipelines, Cloud fabrics)[cite: 5]. It represents a deep harmony between People, Process, and Technology. You can automate resource provisioning, but you cannot automate trust, psychological safety, or shared responsibility[cite: 5].
  • Four Core DevOps Pillars:
    • Cross-functional collaboration to break institutional silos and share risk[cite: 5].
    • Automation workflows to eliminate friction and minimize human error[cite: 5].
    • Fast Feedback implementation via continuous Monitoring and Observability[cite: 5].
    • Continuous Improvement fostered through blameless post-mortems and agile Retrospectives[cite: 5].

Key Takeaways

Design & Systems Mindset

  • Business-First Approach: Engineering goals must always originate from business domain requirements, not just technical preferences[cite: 1, 4].
  • Systems Thinking: Shifting operational focus toward exploring root causes deep beneath the surface instead of patching temporary visual symptoms[cite: 5].
  • Infrastructure Governance: Understanding resource lifecycle controls and structural tagging strategies to ensure optimal cloud financial management.

Technical Architecture

  • Vetting system complexity by hosting Event Storming sessions to establish reliable Bounded Context boundaries[cite: 1, 4].
  • Evaluating compute family capabilities (Compute Spectrum) and configuring decoupled asynchronous message streams.
  • Setting up robust, zero-cost local prototyping environments by merging Cloud Quest tracks with Floci emulators[cite: 3].

Professional Development (Personal Growth)

  • Understanding that confidence and the ability to act despite fear are trainable skills optimized through small habits (5-Second/5-Minute rules)[cite: 2, 4].
  • Embracing shared runtime responsibility, blameless code iterations, and cross-functional transparency driven by DevOps culture[cite: 5].

Applying to Work

  • Conducting Event Storming and DDD: Scheduling interactive discovery workshops with business stakeholders to map out ubiquitous vocabularies and refactor current software architectures[cite: 1, 4].
  • Refactoring Microservices Boundaries: Applying Bounded Context strategies to clean up backend service dependencies, systematically replacing synchronous API blocks with asynchronous queue message handling[cite: 1].
  • Optimizing Local Engineering Loop: Provisioning the open-source Floci emulator on my local workstation to mock Lambda executions, S3 object flows, and DynamoDB schemas prior to AWS deployment, boosting deployment speed[cite: 3].
  • Embedding DevOps Feedback Systems: Setting up proactive code logging frameworks (Fast Feedback loops) and scheduling blameless sprint retrospectives to continuously elevate team execution standards[cite: 5].

  • Deep Practical Grounding: Moving beyond standard high-level cloud slide decks, the seminar provided immense tangible value by introducing open-source local testing ecosystems (Floci) and addressing day-to-day engineer bottlenecks like public speaking anxiety and procrastination habits[cite: 2, 3, 4].
  • Exceptional Network Synergy: The workshop established an outstanding collaborative environment to discuss DevOps cultures, share battle-tested project challenges, and exchange hackathon strategies, fully reinforcing a continuous “Go Build” mindset[cite: 1, 5].
